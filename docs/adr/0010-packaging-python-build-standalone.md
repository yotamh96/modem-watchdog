# ADR-0010 — Packaging via python-build-standalone + uv + custom debhelper

| Field        | Value          |
| ------------ | -------------- |
| Status       | Accepted       |
| Date         | 2026-05-06     |
| Deciders     | Eng team       |
| Closes       | Q8 (Jetson Python); Q6/Q7 ops notes |

## Context

ADR-0001 decided "ship the runtime we tested with" and "do not depend
on the system Python." This ADR fills in the *how*. The constraints
(`.planning/research/STACK.md` §2):

- **Jetson system Python is 3.8.10** (frozen in L4T R35.6.4 /
  Ubuntu 20.04). Patching the system Python is operationally hostile:
  Zao's runtime depends on it.
- **pydantic v2.11+ requires Python ≥3.9.** Using pydantic v1 is not
  acceptable (closed enums, discriminated unions, mypy integration
  all require v2).
- **deadsnakes does not publish Python 3.11+ for Ubuntu 20.04**
  ("focal"). Even if it did, adding a PPA to a fleet of production
  Jetson boxes is an operational risk.
- **`dh-virtualenv`** (the standard Debian tool for venv-in-deb) uses
  the system Python as the venv base. With system Python 3.8.10, it
  produces a 3.8 venv — useless.
- **NFR-43**: reproducible builds. The packaging path must produce
  bit-for-bit identical `.deb` files from the same source tree.
- **NFR-50**: self-contained. The `.deb` must not add apt dependencies
  on `python3-*` packages.
- **NFR-51**: `.deb` size ≤ 40 MiB.
- **NFR-52**: install-time internet not required (the box may be
  offline at first install; C20).

**Closes Q8** (PROJECT.md: Jetson Python runtime): bundle CPython 3.12
via `astral-sh/python-build-standalone`. Also partially addresses
**Q6** (config-change communication) and **Q7** (carrier-table
ownership) — documented in the Consequences section.

## Options considered

### A. python-build-standalone + uv + custom debhelper (chosen)

`astral-sh/python-build-standalone` publishes relocatable CPython
builds for `aarch64-unknown-linux-gnu` with a glibc 2.17 baseline.
Ubuntu 20.04 ships glibc 2.31 — comfortable margin. The tarball is
~30 MiB; the final `.deb` is ≤40 MiB (NFR-51).

### B. deadsnakes PPA

Not available for focal aarch64. Ruled out.

### C. Compile CPython from source in CI

Reproducible but slow (~20 min per build). Not worth the CI cost
when PBS publishes a pinned, pre-built binary we can SHA256-verify.

### D. Write Python 3.8-compatible code

Would require dropping `match` statements (3.10+), `asyncio.TaskGroup`
(3.11+), `tomllib` (3.11+), and parameterized type aliases (3.10+).
Significant velocity loss; defeats the purpose of choosing Python as
the rewrite language. Ruled out.

### E. dh-virtualenv

Uses the system Python. On a Jetson Orin NX with Python 3.8.10, this
produces a 3.8 venv. Ruled out.

## Decision

Use `astral-sh/python-build-standalone` to fetch a relocatable
CPython 3.12.x tarball at build time; install runtime libraries via
`uv pip install --frozen --no-deps -r packaging/requirements.lock`;
package via a **custom `debian/rules`** (NOT `dh-virtualenv`).

The canonical 5-step recipe (from `.planning/research/STACK.md`
§"Packaging recipe"):

```
1. Download pinned PBS tarball; SHA256 in debian/python.sha256; verify
   before unpack. (debian/rules step: python-build-standalone download.)

2. Unpack to debian/<pkg>/opt/spark-modem-watchdog/python/
   — the FINAL install path, not a build-time staging path.
   (PITFALLS §18.3: venv paths bake at unpack time; use the final path.)

3. uv pip install --frozen --no-deps \
       -r packaging/requirements.lock \
       --python debian/<pkg>/opt/spark-modem-watchdog/python/bin/python3.12
   Installs all 9 (10 with pydantic-settings) runtime libraries.

4. python -m compileall \
       --invalidation-mode checked-hash \
       debian/<pkg>/opt/spark-modem-watchdog/
   under SOURCE_DATE_EPOCH (NFR-13 warm cache; NFR-43 reproducibility).
   .pyc files do not bake builder paths (we are already at the final path).

5. Install systemd unit + default carrier-table YAML + libexec scripts.
   debian/postinst smoke-tests all 10 imports under the bundled python.
   ExecStartPre= re-runs the same check at every daemon start (FR-60).
```

### Versioning

- **Releases**: `2.0.0-1` (Debian-style upstream-debian split).
- **Dev builds**: `2.0.0-0.git<short-sha>-1` — visibly distinct from
  releases; never accidentally installed in production.
- **Reproducibility**: `SOURCE_DATE_EPOCH=$(git log -1 --format=%ct)`
  exported in `debian/rules`; `requirements.lock` (output of
  `uv pip compile`) is committed; `uv pip install --frozen` enforced
  in `debian/rules`.

### requirements.lock

`packaging/requirements.lock` is generated by:
```
uv pip compile packaging/requirements.in \
    --python-platform linux \
    --python-version 3.12 \
    -o packaging/requirements.lock
```

`--python-platform linux` excludes `win-inet-pton` and targets the
aarch64/Linux deployment environment. The lockfile is committed to git
(NFR-43). It is regenerated only when `requirements.in` changes.

## Consequences

- **`.deb` size**: ~30–35 MiB (NFR-51 ceiling 40 MiB).

- **Security update story**: rebuild the `.deb` on every CPython
  security release (4–6× per year on stable branches). Carrier-table
  edits are config-only and do not require a rebuild. The rebuild
  policy is an operational discipline, not an automated gate in v2.0.

- **PITFALLS §18.3 (venv path relocation)**: the Python installation
  is unpacked to the FINAL `/opt/spark-modem-watchdog/python/` path
  during build. `compileall` runs under the destination layout so
  `.pyc` files do not bake builder paths. `debian/rules` asserts:
  ```
  python3.12 -c "import sys; assert sys.executable == \
      '/opt/spark-modem-watchdog/python/bin/python3.12'"
  ```
  post-install as a smoke-test gate.

- **PITFALLS §18.5 (certifi rotation)**: a stale `.deb` ships a stale
  certifi bundle, which eventually breaks TLS to webhook endpoints.
  Mitigation is operational: rebuild policy (see above). A fleet-wide
  certifi rotation without a full rebuild is deferred to v2.1.

- **Q6 ops note** (config-change communication): configuration
  precedence is `CLI flags > env vars > /etc/spark-modem-watchdog/
  conf.d/*.yaml > defaults`. Implemented in `src/spark_modem/config/
  Settings` (Plan 06, Phase 1). SIGHUP transactional reload for
  data-only fields (APN table, webhook URL, thresholds) lands in
  Phase 3. Topology-affecting fields (`state_root`, `run_dir`,
  `metrics_socket_path`) are tagged `RELOAD_RESTART` — the SIGHUP
  reload refuses to apply them mid-flight and emits a
  `config_reload_requires_restart` event with the field names.

- **Q7 ops note** (carrier-table ownership): the carrier table ships
  at `debian/conf.d/00-carriers.yaml` (12 entries day-one: IL
  verified; US/UK/DE marked `unverified: true`). **Ownership**: product
  owns the table contents; engineering ships via `.deb`. Operators add
  new MCC/MNC entries via `/etc/spark-modem-watchdog/conf.d/*.yaml`
  overlay files (FR-33) — no rebuild required. SIGHUP or daemon
  restart picks up the overlay. New carrier entries in the default
  table require a `.deb` release.

## Risks and mitigations

| Risk | Mitigation |
| ---- | ---------- |
| PBS upstream stops publishing Python 3.12 builds | Pin a known-good release tag in `debian/python.sha256`; pre-download the tarball to fleet artifact storage before the GitHub source is retired. |
| glibc compatibility regression | PBS's glibc 2.17 baseline is well below Ubuntu 20.04's 2.31. CI builds on a real aarch64 runner (CONTEXT.md B-01) catch this before it reaches the fleet. |
| Build-time download flake | `debian/rules` caches the PBS tarball locally by SHA256. Subsequent rebuilds reuse the cached tarball if the SHA matches. CI runners use a persistent cache volume. |
| `uv pip install` picks up a non-locked dependency | `--frozen` flag on `uv pip install` enforces the lockfile exactly; any drift fails the build. `requirements.lock` is committed and reviewed on every PR that changes `requirements.in`. |
| `.deb` postinst import smoke-test fails silently | Non-zero exit from `postinst` fails `apt install`. The CI pipeline runs `apt install` in a fresh container after `dpkg-buildpackage` and checks the exit code. |

## Implementation reference

- `debian/rules` — custom debhelper build recipe (Plan 02).
- `debian/python.sha256` — pinned PBS tarball SHA256 (Plan 02).
- `debian/spark-modem-watchdog.postinst` — import smoke-test (Plan 02).
- `scripts/postinst_smoke_test.sh` — shared smoke-test script called
  from both `postinst` and `ExecStartPre=` (Plan 02).
- `scripts/build_deb.sh` — local developer build helper (Plan 02).
- `.github/workflows/build-deb.yml` — CI build on self-hosted aarch64
  runner (Plan 02; CONTEXT.md B-01).
- `packaging/requirements.in` + `packaging/requirements.lock` —
  dependency manifest + lockfile (Plan 01; updated in Plan 06 to add
  `pydantic-settings`).

## Revisit when

- CPython 3.13's free-threaded (GIL-optional) story stabilizes and we
  want concurrent GIL-free execution. Then re-evaluate PBS's 3.13
  aarch64 build quality and the aarch64 wheel ecosystem.
- Sierra publishes a Linux ABI shift that requires a newer glibc baseline.
  Then re-evaluate whether PBS's glibc 2.17 baseline still provides
  sufficient margin.
- JetPack ships a system Python ≥3.12 (i.e. Jetson updates Ubuntu
  version or system Python version). Then "do we still bundle?" is
  worth revisiting — the answer may still be "yes" for reproducibility.
- The `.deb` size grows above 35 MiB. Then audit which library is
  unexpectedly large and consider stripping debug symbols or splitting
  into `spark-modem-watchdog` + `spark-modem-watchdog-libs` packages.
