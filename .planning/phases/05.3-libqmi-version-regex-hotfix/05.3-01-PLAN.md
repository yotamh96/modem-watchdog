# Plan 05.3-01 — Accept `qmicli X.Y.Z` in libqmi version regex

## Objective

Broaden `_LIBQMI_VERSION_RE` in `src/spark_modem/qmi/version.py` to match
either the `qmicli X.Y.Z` first-line banner OR the `Compiled with
libqmi-glib X.Y.Z` footer. Capture the JetPack 5.1.5 / libqmi 1.30.4
`qmicli --version` output as a test fixture and assert the new regex parses
it correctly.

## Must-haves

### Truths
- `_LIBQMI_VERSION_RE` matches both `qmicli 1.30.4\n...` (no footer) AND the
  existing fixture forms with a `libqmi-glib X.Y.Z` line.
- A new fixture file `tests/fixtures/qmicli/version/1.30/jetpack-1.30.4.txt`
  contains the bench Jetson's exact stdout (with the leading comment header
  documenting source + capture date).
- A new test `test_detect_libqmi_version_parses_jetpack_qmicli_only_format`
  asserts `await detect_libqmi_version() == "1.30.4"` against the new fixture.
- All existing tests in `tests/unit/qmi/test_version.py` continue to pass
  (the new regex returns the same version on the existing fixtures because
  `qmicli X.Y.Z` and `libqmi-glib X.Y.Z` are lockstep in upstream libqmi).
- `mypy --strict` and `ruff check` are green on the modified files.

### Artifacts
- path: src/spark_modem/qmi/version.py
  provides: broadened libqmi version regex
  contains: `(?:qmicli|libqmi-glib)\s+(\d+\.\d+\.\d+)`
- path: tests/fixtures/qmicli/version/1.30/jetpack-1.30.4.txt
  provides: bench Jetson qmicli --version output fixture
  contains: 'qmicli 1.30.4'
- path: tests/unit/qmi/test_version.py
  provides: new regression test for qmicli-only format
  contains: 'test_detect_libqmi_version_parses_jetpack_qmicli_only_format'

## Tasks

### Task 1 — Broaden the regex

Edit `src/spark_modem/qmi/version.py` line 29-31. Replace:

```python
_LIBQMI_VERSION_RE: Final[re.Pattern[str]] = re.compile(
    r"libqmi-glib\s+(\d+\.\d+\.\d+)", re.IGNORECASE
)
```

with a regex that matches either `qmicli` or `libqmi-glib` as the leading
token, plus an inline comment explaining the JetPack `qmicli`-only format
and the lockstep-versioning rationale.

### Task 2 — Add the bench fixture

Create `tests/fixtures/qmicli/version/1.30/jetpack-1.30.4.txt` with the
bench Jetson's exact stdout. Include a leading `# ...` comment header naming
the source (bench Jetson Orin NX, JetPack 5.1.5, Ubuntu 20.04), the capture
date, and the distinguishing trait (NO libqmi-glib footer).

### Task 3 — Add the regression test

Insert a new test function in `tests/unit/qmi/test_version.py` between
`test_detect_libqmi_version_parses_1_30` and `test_detect_libqmi_version_parses_1_32`.
The function loads the new fixture and asserts the parsed version is
exactly `"1.30.4"`. Docstring should reference Phase 05.3 and the
2026-05-12 bench deploy.

### Task 4 — Verify

```bash
uv run mypy --strict src/spark_modem/qmi/version.py
uv run ruff check src/spark_modem/qmi/version.py tests/unit/qmi/test_version.py
uv run pytest tests/unit/qmi/test_version.py -q
```

All three must exit green; pytest must report ≥11 passed.
