       """Phase 3 unit-file audit — parses the .service file and asserts directives.

       This test is the regression gate for U-01..U-05 + NFR-30. If a
       future merge accidentally drops WatchdogSec= or sets

       Cross-platform — pure file parsing, no systemd interaction.

       linux_only to this file (Issue #6 RESOLVED).

       from __future__ import annotations

       from pathlib import Path

       import pytest


       _UNIT_PATH = Path(__file__).resolve().parents[2] / "debian" / "spark-modem-watchdog.service"
       _LOGROTATE_PATH = Path(__file__).resolve().parents[2] / "debian" / "spark-modem-watchdog.logrotate"


       def _read_unit() -> dict[str, list[str]]:
           text = _UNIT_PATH.read_text(encoding="utf-8")
           directives: dict[str, list[str]] = {}
           for line in text.splitlines():
               stripped = line.strip()
               if not stripped or stripped.startswith("#") or stripped.startswith("["):
                   continue
               if "=" not in stripped:
                   continue
               key, _, value = stripped.partition("=")
               directives.setdefault(key.strip(), []).append(value.strip())
           return directives


       @pytest.fixture(scope="module")
       def unit_directives() -> dict[str, list[str]]:
           return _read_unit()


       def test_type_notify(unit_directives: dict[str, list[str]]) -> None:
           assert unit_directives.get("Type") == ["notify"]

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| systemd → daemon (unit directives) | The .service file is the authoritative declaration of how systemd starts and supervises the daemon. A regression that drops NoNewPrivileges= or grants extra caps is a security incident; the audit test is the regression gate. |
| Logrotate (root cron) → daemon | Logrotate runs as root via cron and recreates events.jsonl with `create 0640 root adm`. Empty postrotate (R-02) means the daemon detects rotation via inotify and reopens; no inter-process signal needed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08-01 | Tampering | Unit file regression dropping a hardening directive | mitigate | test_unit_file_audit.py pins every U-01..U-05 directive; runs cross-platform on every PR. |
| T-03-08-02 | DoS | Default StartLimit (5 restarts in 10s) bricks fleet on bad config rollout (PITFALLS §4.2) | mitigate | U-02 overrides: StartLimitIntervalSec=300, StartLimitBurst=20, RestartSec=10; ExecStartPre=config-check (U-05) catches bad configs BEFORE main process. |
| T-03-08-03 | Tampering | LoadCredential silent failure on systemd 245 (PITFALLS §4.3) | mitigate | U-03 explicitly omits PrivateMounts/PrivateTmp; daemon refuses to start when webhook_signing_secret_required=true and credential file missing/empty (Phase 1 wired). |
| T-03-08-04 | Tampering | RuntimeDirectory cleaned on stop, taking out PID lock + marker (PITFALLS §4.4) | mitigate | RuntimeDirectoryPreserve=yes (load-bearing per U-03); test_runtime_directory_preserve_yes pins the directive. |
| T-03-08-05 | Elevation of Privilege | Future PR adding suid binary or extra capability | mitigate | NoNewPrivileges=yes pinned by audit test; CapabilityBoundingSet caps at the four enumerated values; any addition flagged by audit. |
| T-03-08-06 | DoS | Logrotate triggers reopen storm via non-empty postrotate signaling | mitigate | R-02 EMPTY postrotate; daemon's inotify-driven reopen handles rotation autonomously without external signal; verified by test_logrotate_postrotate_empty. |
</threat_model>
