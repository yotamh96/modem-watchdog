### WR-01: `sd.ready()` skipped on cycle-0 crash, never fires on subsequent recovery

**Files modified:** `src/spark_modem/daemon/main.py`
**Commit:** f21b88c
**Applied fix:** Replaced the `if cycle_id == 0` predicate in `_cycle_loop`'s
step 6 with a `ready_fired_ref: list[bool] = [False]` single-cell list (same
pattern as `cycle_count_ref`). After a successful cycle, READY=1 fires iff
`not ready_fired_ref[0]`, then `ready_fired_ref[0]` flips to True. New
semantics match the stated operational intent: "fire READY on the FIRST
SUCCESSFUL cycle", so a cycle-0 driver crash followed by cycle-1 success
still fires READY (previously cycle 1 would skip the predicate and the
daemon would stay in systemd's "starting" state until TimeoutStartSec
killed it).

The fix is conservative — when cycle 0 succeeds (the happy path the existing
integration test covers), the assertion `len(fake_sd.ready_calls) == 1`
still holds because the predicate only fires once. The behaviour change is
strictly additive on the cycle-0-crash + cycle-1-success branch.

### WR-02: Private-attribute access into `WebhookPoster._dns_cache` from `_production_main`

**Files modified:** `src/spark_modem/webhook/poster.py`, `src/spark_modem/daemon/main.py`
**Commit:** f3ef66f
**Applied fix:** Added a read-only `@property dns_cache` on `WebhookPoster`
that returns `self._dns_cache`, and updated the `SighupSwapper`
construction in `_production_main` to consume `webhook_poster.dns_cache`
instead of `webhook_poster._dns_cache`. The property is the minimum-diff
fix from the review (the alternative — pass `dns_cache` into both
constructors — would have required a wider plumbing change). W-02's
SIGHUP-driven DNS force-refresh on `webhook_url` change is now mediated
through a public surface that a future refactor of `WebhookPoster` cannot
silently break without breaking type-checked callers first.
