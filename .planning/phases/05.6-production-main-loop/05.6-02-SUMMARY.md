## Decisions Made

- **`cast(Any, ...)` at producer-boundary call sites and `supervisor_event_logger: Any` alias.** Each producer module declares its own narrow `_EventQueueProto.put_nowait(item: object)` Protocol co-located with the producer (to avoid cross-package imports into `event_sources/`). Python Protocols are method-parameter-invariant; the concrete `asyncio.Queue[WakeSignal].put_nowait(item: WakeSignal)` is mypy-invariant-incompatible with `put_nowait(item: object)`. Same situation for the asyncinotify producer's local `_ZaoTailerProto` vs the canonical `ZaoLogTailer` Protocol, and for the supervisor's `EventLogWriterProto.append(event: object)` vs the concrete `EventLogWriter.append(event: Event-union)`. **Resolution:** `cast(Any, event_queue)` / `cast(Any, zao_tailer)` at 5 producer-boundary call sites + one `supervisor_event_logger: Any = event_logger` alias for the 4 `restart_on_crash` `event_logger=` kwargs. The runtime contract is correct (producers only ever put `WakeSignal` members onto the queue, which is a narrower argument than the queue accepts); the cast just makes the type-erasure explicit at the boundary. Documented at the cast site.
- **Supervised wrapper coroutines re-check `*_factory is not None` on every call** (not pre-computing `*_coro` coroutine objects). Per the plan's action body Important note: an already-awaited coroutine raises `RuntimeError`, so each supervised wrapper must construct a fresh inner coroutine on each `restart_on_crash` re-entry. The wrapper body branches on the factory parameter and awaits either the test factory or the production-default helper.
- **`producer_tasks` list declared OUTSIDE the TaskGroup body.** Plan 05.6-04's `_sigterm_watcher` closure (defined OUTSIDE the TaskGroup too) closes over `producer_tasks` and reads it once `shutdown_event` fires. Declaring inside the TaskGroup would scope the list to the TaskGroup body and the choreography couldn't see it.
- **Re-derived `_MainIssueEmitterProto` locally** instead of importing `kmsg_producer._IssueEmitterProto`. The underscore prefix marks the producer's Protocol as module-private; re-deriving inline keeps the cross-module import surface clean (the local Protocol has the same shape: `def emit_host_issue(self, *, detail: IssueDetail, raw_line: str) -> None`).
- **`_LoggingHostIssueEmitter` placeholder logs at INFO** (with `raw_line[:200]` truncation per T-05.6-02-01 mitigation) so the journal records each detected host issue and the bench Jetson can observe FR-14 detection working even before plan 05.6-03 wires the cycle-driver-fed accumulator.

**1. [Rule 3 - Blocking] mypy Protocol invariance at producer boundaries**

- **Found during:** Task 2, first `mypy --strict` pass after writing the producer call sites.
- **Issue:** 9 mypy errors at 5 producer-boundary call sites + 4 `restart_on_crash` `event_logger=` kwargs. Each producer module declares its own narrow `_EventQueueProto.put_nowait(item: object)` Protocol co-located with the producer (the producer modules deliberately avoid importing from `event_sources/supervisor` or `wire/state` to keep cross-package imports out of `event_sources/`). Python Protocols are method-parameter-invariant by default; the concrete `asyncio.Queue[WakeSignal].put_nowait(item: WakeSignal)` and the production `ZaoLogInotifyTailer` (different nominal Protocol than asyncinotify's local `_ZaoTailerProto`) are mypy-invariant-incompatible with the narrow Protocols. Same situation for `EventLogWriterProto.append(event: object)` vs concrete `EventLogWriter.append(event: Event-union)`.
- **Fix:** `cast(Any, event_queue)` / `cast(Any, zao_tailer)` at 5 producer-boundary call sites + one `supervisor_event_logger: Any = event_logger` alias for the 4 `restart_on_crash` `event_logger=` kwargs. Added `Any, cast` to the `typing` import block. The cast site has an inline comment explaining the type-erasure rationale.
- **Why this is correct:** The producers only ever call `put_nowait(WakeSignal.UDEV)` etc., which is a NARROWER argument to the queue's `put_nowait(WakeSignal)`. The runtime contract is sound; the cast just acknowledges that the producer modules' Protocols are deliberately structural (the wire boundary uses the broader `object` so cross-package imports stay disciplined).
