## Decisions Made

- **`ge=1.0` honored over plan's 0.1/0.5 acceptance examples.** The plan's action body, threat model (T-05.6-01-03), CONTEXT.md C-02, and PATTERNS.md all converged on `ge=1.0`. Two of the plan's acceptance criteria sub-cases (`SPARK_MODEM_CYCLE_INTERVAL_SECONDS=0.1` env-override and `SPARK_MODEM_CYCLE_INTERVAL_SECONDS=0.5` manual smoke test) contradict that constraint. Honored the constraint; tested env-override with 1.0 and 2.5 instead. The threat model explicitly cites `ge=1.0` as a defense against operator-induced DoS, which would be undermined by lowering the bound.
- **`# noqa: F401` for keep-alive imports.** The plan required deleting all `_ = restart_on_crash | _ = UdevInventory | _ = ZaoLogInotifyTailer | _ = signal` lines (acceptance criterion #6 == 0) while keeping the imports themselves alive ("later plans need them"). The natural Python idiom for that is `# noqa: F401`, applied to the three imports that this plan does not call.
- **Module-level `_stub_*` helpers.** The plan's action body uses inner async closures inside `_production_main`. Inline they pushed the function past ruff's PLR0915 (50-statement) budget. Factoring the three helpers out to module scope (taking their dependencies as keyword arguments) reduced the function's statement count and matches Phase 3 L-01's "helpers move out when they grow" convention. PLR0915 noqa applied for the residual 51-statement count that plans 02-04 will grow regardless.
- **`asyncio.wait(FIRST_COMPLETED)` in `_stub_sighup_watcher`.** The plan's literal `while not shutdown_event.is_set(): await sighup_event.wait()` is a deadlock: when SIGTERM fires without SIGHUP, the inner `await sighup_event.wait()` blocks forever and the TaskGroup never unwinds. Caught during my own smoke test (timeout 10 / rc=124). Replaced with `asyncio.wait({shutdown_task, sighup_task}, return_when=FIRST_COMPLETED)` so the watcher exits promptly under either trigger.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `_stub_sighup_watcher` deadlock on SIGTERM-without-SIGHUP**
- **Found during:** Task 3, post-implementation smoke test (timeout 10 / rc=124 on Windows-side asyncio loop).
- **Issue:** The plan's literal sighup-watcher body — `while not shutdown_event.is_set(): await sighup_event.wait(); sighup_event.clear(); …` — wedges on the inner `await sighup_event.wait()` when shutdown_event is set without sighup_event ever firing. The TaskGroup then hangs in cleanup until the OS kills the daemon.
- **Fix:** Replaced with `asyncio.wait({shutdown_task, sighup_task}, return_when=FIRST_COMPLETED)`, branching on which task completed, and cleaning up pending tasks in a `finally` block.
- **Files modified:** `src/spark_modem/daemon/main.py` (`_stub_sighup_watcher` function body, lines ~234-258).
- **Verification:** Smoke test passes (rc=0; status.json written; cycle_index increments).
