# Plan 05.5-01 — Retry firmware probe on transient qmi-proxy failures

## Objective

Wrap `compute_fleet_triple`'s firmware probe in a 3-attempt retry loop
with 0.5s backoff. Check `cp.exit_code` before parsing; if non-zero,
capture `cp.stderr` and retry. On exhausted retries, raise
`QmiVersionDetectionFailed` with a clear message that names the
underlying cause (qmicli stderr OR parser error).

## Must-haves

### Truths
- `compute_fleet_triple` retries on qmicli non-zero exit AND on parser
  UNEXPECTED_OUTPUT / revision=None (any failure mode that could be
  transient under qmi-proxy contention).
- The retry budget is `_FIRMWARE_PROBE_ATTEMPTS = 3` attempts with
  `_FIRMWARE_PROBE_BACKOFF_S = 0.5` seconds between attempts.
- On exhausted retries, the raised `QmiVersionDetectionFailed` message
  contains both the literal `"failed after 3 attempts:"` AND the
  underlying cause (qmicli stderr excerpt OR parser error).
- `detect_libqmi_version` is NOT modified — it doesn't use the proxy.
- `_FakeWrapper` in tests supports either single-stdout or a
  responses-sequence shape; the sequence shape exhausts and repeats
  the final entry so over-call doesn't IndexError.
- Two new tests cover (a) transient-failure-then-success and (b)
  all-attempts-fail-with-stderr-captured.
- The existing `test_compute_fleet_triple_firmware_qmierror_raises`
  match string is updated to reflect the new error wording (semantic
  intent preserved).
- `mypy --strict`, `ruff`, and `pytest tests/unit/qmi/` all green;
  pytest reports ≥98 passed (up from 95 — 1 updated + 2 new).

## Tasks

### Task 1 — Add the retry-loop infrastructure

Edit `src/spark_modem/qmi/version.py`:
- Add `import asyncio` to the import block (for `asyncio.sleep`).
- Add module-level constants:
  - `_FIRMWARE_PROBE_ATTEMPTS: Final[int] = 3`
  - `_FIRMWARE_PROBE_BACKOFF_S: Final[float] = 0.5`
- Inline a comment block explaining the bench-Jetson contention pattern
  and the 25% / 99% reasoning behind the chosen budget.

### Task 2 — Replace the single-shot probe with a retry loop

Replace the one-shot `cp = await wrapper.dms_get_revision()` + parser
block with a `for attempt in range(_FIRMWARE_PROBE_ATTEMPTS):` loop:
- If `cp.exit_code != 0`: stash `last_error = "qmicli exit_code=N stderr={excerpt}"` and continue.
- Else if `parsed` is `QmiError`: stash `last_error = "parser returned QmiError: ..."` and continue.
- Else if `parsed.revision is None`: stash `last_error = "parser returned revision=None ..."` and continue.
- Else: break out of the loop with `revision = parsed.revision`.
- Between attempts (NOT after the last one), `await asyncio.sleep(_FIRMWARE_PROBE_BACKOFF_S)`.
- After the loop, if `revision is None`, raise `QmiVersionDetectionFailed(f"dms_get_revision failed after {_FIRMWARE_PROBE_ATTEMPTS} attempts: {last_error}")`.

### Task 3 — Extend `_FakeWrapper` to support response sequences

In `tests/unit/qmi/test_version.py`, broaden `_FakeWrapper.__init__` to
accept either the existing single-stdout / single-exit-code shape OR a
new `responses: list[tuple[int, bytes, bytes]]` parameter. Add a
`call_count` property so tests can assert how many attempts were used.
Keep the single-stdout path working unchanged (all existing tests
continue to construct `_FakeWrapper(stdout=...)`).

### Task 4 — Add two retry-behavior tests

Insert at the bottom of `tests/unit/qmi/test_version.py`:

- `test_compute_fleet_triple_retries_transient_qmicli_failure` — first
  two attempts return `(exit_code=1, stdout=b"", stderr=<CID-fail
  message>)`, third returns `(0, fw_body, b"")`. Asserts the triple
  parses and `wrapper.call_count == 3`.

- `test_compute_fleet_triple_exhausts_retries_with_stderr_in_error` —
  every attempt returns `(1, b"", <CID-fail stderr>)`. Asserts
  `QmiVersionDetectionFailed` is raised with a message containing
  `"failed after 3 attempts"` AND `"CID allocation failed"`, and
  `wrapper.call_count == 3`.

### Task 5 — Update the existing parser-error test

The existing `test_compute_fleet_triple_firmware_qmierror_raises` match
regex `r"dms_get_revision returned QmiError"` no longer fits the new
wording. Update it to `r"dms_get_revision failed after 3 attempts: parser returned QmiError"`.
The test still validates the same semantic (malformed stdout → exception).

The .deb built from a commit containing this fix must install on the
bench Jetson and the daemon's preflight step must advance past the
