# Plan 05.4-01 — Accept singular `Device revision retrieved` header

## Objective

Update `parse_get_revision` to accept both `Device revisions retrieved`
(plural) and `Device revision retrieved` (singular). libqmi emits the
plural form when both `Revision:` and `Boot code:` lines are present, and
the singular form when only `Revision:` is present. The bench Jetson's
SWI9X50C modems emit the singular form.

## Must-haves

### Truths
- The substring check `"Device revisions retrieved" not in body` is
  replaced with a regex match against `Device revisions? retrieved`.
- A new fixture `tests/fixtures/qmicli/get_revision/1.30/jetpack-singular.txt`
  captures the bench Jetson's exact `qmicli --dms-get-revision` output
  with a leading comment header naming source / modem / capture date.
- A new test `test_parser_accepts_singular_revision_header_jetpack`
  asserts the new fixture parses to the expected SWI9X50C revision
  string, including the build-id + jenkins build path.
- All existing tests in `tests/unit/qmi/parsers/test_get_revision.py`
  continue to pass (the regex `revisions?` still matches the plural
  form used by the existing test fixtures).
- `mypy --strict`, `ruff`, and `pytest` all green.

### Artifacts
- path: src/spark_modem/qmi/parsers/get_revision.py
  provides: regex-based response-header match
  contains: 'Device revisions? retrieved'
- path: tests/fixtures/qmicli/get_revision/1.30/jetpack-singular.txt
  provides: bench Jetson singular-revision fixture
  contains: 'Device revision retrieved'
- path: tests/unit/qmi/parsers/test_get_revision.py
  provides: regression test for singular-revision header
  contains: 'test_parser_accepts_singular_revision_header_jetpack'

## Tasks

### Task 1 — Replace the substring header check with a regex

Edit `src/spark_modem/qmi/parsers/get_revision.py`:
- Remove `_RESPONSE_HEADER: Final[str] = "Device revisions retrieved"`
- Add `_RE_RESPONSE_HEADER: Final[re.Pattern[str]] = re.compile(r"Device revisions? retrieved")`
- Change the early-return check from `if _RESPONSE_HEADER not in body:`
  to `if _RE_RESPONSE_HEADER.search(body) is None:`
- Update the module docstring to document BOTH the plural form (with Boot
  code line) and the singular form (Revision only), with rationale.

### Task 2 — Add the bench Jetson fixture

Create `tests/fixtures/qmicli/get_revision/1.30/jetpack-singular.txt`
with a leading `#`-comment header (source = bench Jetson Orin NX,
JetPack 5.1.5, Ubuntu 20.04, libqmi 1.30.4; modem = SWI9X50C; capture
date = 2026-05-12; distinguishing trait = singular header + no Boot code
line). Body matches the bench Jetson output byte-for-byte: the tab-
indented `Revision: 'SWI9X50C_01.14.03.00 b06bd3 jenkins 2020/09/23 10:53:35'`
line with a trailing newline.

### Task 3 — Add the regression test

Insert `test_parser_accepts_singular_revision_header_jetpack` between
`test_parser_accepts_libqmi_1_30` and `test_parser_accepts_libqmi_1_32_fixture`
in `tests/unit/qmi/parsers/test_get_revision.py`. The test loads the new
fixture, calls `parse_get_revision`, and asserts the result is a
`GetRevisionResult` with `revision` equal to the full SWI9X50C revision
string (preserving the build-id + jenkins build path).

### Task 4 — Verify

```bash
uv run mypy --strict src/spark_modem/qmi/parsers/get_revision.py
uv run ruff check src/spark_modem/qmi/parsers/get_revision.py tests/unit/qmi/parsers/test_get_revision.py
```
